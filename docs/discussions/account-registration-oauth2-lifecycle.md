# 账号注册、OAuth2 与账号生命周期结论

本文记录注册策略、OAuth2 首次登录、迁出/注销状态和提示边界的公开讨论结论。后端内部表结构、迁移和接口草案不在本文展开。

## 注册策略

注册策略是实例级配置，不写死。

注册入口可以配置为：

- 关闭注册：只能管理员创建账号。
- 邀请码注册：用户持邀请码注册。
- 公开注册：用户自行注册。

注册校验可以配置：

- 是否必须邮箱验证。
- 是否启用验证码或人机校验。
- 同 IP 注册频率限制。
- 邮箱域名白名单：为空表示不限制；非空时只允许精确匹配的域名注册。
- 是否需要管理员审核后启用。

默认资源限制包括：

- 新账号默认项目数量配额。
- 文件、附件、本地上传声音等真实文件资源的容量配额。
- 迁移频率限制，例如每 X 分钟/小时/天/月最多 N 次。

迁移不单独设置启用开关。本机 -> 服务端 只看目标配额和迁移频率；服务端 -> 本机 每账号只允许一次。

## 登录页与注册入口

- 注册入口只出现在选定业务入口之后，不在未选择服务端时展示通用注册表单。
- 客户端先读取该业务入口公开的登录方式和注册策略，再决定显示账号密码登录、OAuth2 登录、邀请码注册、公开注册或“联系管理员”提示。
- 关闭注册时，不显示自行注册表单；可以提示该服务端需要管理员创建账号。
- 邀请码注册时，注册表单必须包含邀请码；邀请码校验失败时停留在注册表单并显示原因。
- 公开注册时，注册表单展示邮箱、密码和该服务端要求的验证码、人机校验或邮箱验证步骤。
- 邮箱域名白名单命中失败时，前端显示该服务端不接受该邮箱域名注册，不建议用户反复提交。
- 需要管理员审核时，注册成功后进入等待审核状态，不直接进入主界面。
- 登录成功但账号状态不是 `active` 时，不进入主界面，按账号状态展示原因。

## 邀请码

邀请码可以配置：

- 可用次数。
- 有效期。
- 对应默认配额。
- 自己的邮箱域名白名单。
- 是否需要管理员审批激活。

邀请码的邮箱白名单优先于实例默认白名单。管理员手动创建账号可以不受白名单限制，但需要明确提示或二次确认。

## OAuth2 首次登录

OAuth2 首次登录不直接自动创建用户，只进入待绑定或待注册流程，并且必须走同一套注册限制。

- OAuth2 返回邮箱，且后端已有同邮箱账号：提示用户绑定到已有账号。绑定前必须证明账号归属，例如输入密码、邮箱验证码或已登录状态确认。用户也可以改走新账号注册表单，但不能使用已被占用邮箱。
- OAuth2 返回邮箱，且后端没有同邮箱账号：显示注册表单，字段包含邮箱和密码，邮箱默认填 OAuth2 返回值。
- OAuth2 没有返回邮箱：显示注册表单，用户填写邮箱和密码。

补充规则：

- OAuth2 首次接入不能绕过关闭注册、邀请码、邮箱白名单、审核和风控。
- 新账号必须有邮箱和密码，避免 OAuth2 不可用时无法登录。
- OAuth2 绑定成功后只是账号的一种登录方式，不是独立账号。

OAuth2 前端流程：

- 用户先选择业务入口，再点击该入口支持的 OAuth2 登录方式。
- OAuth2 回来后，如果需要绑定已有账号，展示绑定确认页或 `UModal`，要求用户证明已有账号归属。
- OAuth2 回来后，如果需要注册新账号，展示注册表单；有返回邮箱时默认填入，但仍允许用户修改成未占用且符合策略的邮箱。
- 如果 OAuth2 返回邮箱已被占用，注册新账号不能继续使用该邮箱。
- OAuth2 流程中遇到关闭注册、邀请码缺失、邮箱白名单不通过、需要审核或风控限制时，显示对应原因，不自动创建账号。
- OAuth2 绑定或注册成功后，回到正常登录完成流程；如果账号需要审核，停在等待审核状态。

## 账号状态

账号至少区分：

- `active`：正常。
- `disabled`：管理员禁用，可恢复。
- `migrated_out`：已迁出到本机，不可恢复。
- `closed`：已注销，不可恢复。
- `deleting`：删除流程中。
- `deleted`：个人数据删除完成，只保留最小账号留痕。

`disabled` 和 `migrated_out` 不能混用。管理员禁用可恢复，迁出不可恢复。

账号状态的前端处理：

- `active` 可以正常登录和进入主界面。
- `disabled` 不进入主界面，提示账号已被管理员禁用，可以联系管理员处理。
- `migrated_out` 不进入主界面，提示该服务端账号已迁出到本机，不可继续使用。
- `closed`、`deleting`、`deleted` 不进入主界面，提示账号已注销或删除流程中。
- 已迁出或已注销账号不提供重新激活入口。

## 迁出和注销

- 迁出：个人数据导入本机，然后服务端账号进入 `migrated_out`。
- 注销：个人数据删除，不导出到本机，然后账号进入 `closed` 或删除流程。
- 两者都不可恢复。
- 两者都只删除个人数据，不删除需要保留上下文的公开事项、协作记录、空间历史、审计记录等。
- 所有未被删除的数据里，原用户统一显示为 `已注销用户`。
- 系统保留邮箱等基础账号信息或邮箱 hash，用于禁止重复注册和安全审计。

迁出或注销的前置阻塞：

- 如果用户是公开主页唯一负责人，必须先转让或关闭主页。
- 如果用户是空间唯一管理员，必须先转让或解散空间。

公开事项不阻塞迁出或注销。公开事项可以保留，发布人显示为 `已注销用户`；如果没有其他可管理者，后续按公开事项规则自然结束或不可编辑。

## 确认提示

迁出和注销确认页由固定提示和条件提示组成。

固定提示：

- 个人数据会被迁出或删除。
- 服务端账号不可恢复。
- 系统会保留邮箱等基础账号信息，用于禁止重复注册和安全审计。

条件提示：

- 有公开权限或发布过公开内容时，才提示公开事项、公开发布记录不会删除，身份会显示为 `已注销用户`。
- 是公开主页负责人时，提示必须先转让或关闭主页。
- 有协作记录时，提示协作记录不会删除，身份会显示为 `已注销用户`。
- 有空间关系时，提示空间历史不会删除，身份会显示为 `已注销用户`。
- 是空间唯一管理员时，提示必须先转让或解散空间。

没有公开权限、没有发布公开内容的账号，不显示公开事项相关警告。

## 账号设置入口

- 注销账号入口放在 `设置 -> 安全` 或账号安全区，不放在头像菜单。
- 迁出到本机入口仍放在 Desktop Full 的 `设置 -> 数据与来源 -> 服务端业务来源详情 -> 迁出到本机`，不和普通注销混在一起。
- 注销账号是高风险流程，使用独立确认页或宽 `UModal`；必须展示固定提示、条件提示和前置阻塞。
- 前置阻塞未处理完时，禁用最终确认按钮，并提供跳转到公开主页、空间或相关设置的入口。
- 没有公开权限、没有发布公开内容时，注销确认不展示公开事项相关警告。
- 用户完成注销后退出当前服务端会话；客户端中该业务来源不可继续使用，需要移除连接记录或返回登录页。
