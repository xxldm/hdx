# 多来源客户端行为结论

本文记录多来源显示、保存位置和通知偏好的公开讨论结论。长期迁移边界见 ADR 0019；后端内部接口、数据模型和实现切片不在本文展开。

## 来源显示

- 客户端来源平级，不设主账号/次账号。
- 本机模式也允许连接多个服务端和公开来源。
- 主导航按业务模块组织，不按来源或空间拆分。
- 列表默认合并展示所有可用来源，来源和空间只做轻量标签、筛选和状态信息。
- 右上角提供来源状态入口；全部正常时可以弱显示或不显示，有来源离线/登录失效/迁出时显示状态标识。
- 来源状态面板列出来源名称、在线状态、登录状态、上次同步时间、可重试动作和已迁出说明。

## 来源不可用

- 某个来源不可用时，不打断整个页面，其他来源继续展示和操作。
- 已缓存内容可以只读展示，并标记可能不是最新。
- 不可用来源不允许写操作，例如完成、编辑、删除、加入日程或修改通知设置。
- 当前筛选只看不可用来源时，页面显示该来源不可用空态；如果没有缓存，则显示无法加载。
- 匿名公开来源不可用时轻处理，公开流少显示对应内容即可，不做大面积报错。

## 默认保存位置

- 客户端有一个全局 `默认保存位置`。
- 普通新增、首页快捷新增、公开事项加入个人 Todo / 日程，默认保存到该位置。
- 选择来源时可以勾选 `设为默认保存位置`；不勾选只对本次有效。
- 默认保存位置不可写、离线或权限失效时，操作时重新选择。
- 空间上下文和规则生成内容不使用全局默认保存位置。
- 只读来源不能作为保存目标。

## 通知中心与来源

- 通知中心默认合并展示所有来源的站内通知。
- 每条通知带轻量来源标识，并支持按来源筛选。
- 未读角标显示所有来源的未读总数。
- 离线来源只显示本地已有缓存通知，不拉取新通知。
- 点击通知时，来源在线且内容可访问则打开对应内容；来源离线则提示暂时不可用；来源已迁出/已注销则只展示通知文本。
- 强制系统通知可以进入合并收件箱，并带来源标识。

## 通知偏好

- 客户端有一套全局通知偏好，作为所有来源的默认值。
- 新增服务端来源时默认跟随客户端全局偏好。
- 每个来源可以选择跟随客户端默认或自定义此来源。
- 具体事项仍支持 `default` / `custom`：`default` 跟随该来源当前有效默认值，`custom` 使用事项自己的通知设置。
- 客户端设备静音是设备级设置，覆盖所有来源的声音、桌面通知和横幅，不影响站内收件箱。
- 系统关键通知最多只能强制进入站内收件箱，不能强制声音、桌面通知或弹出式横幅。
- Toast 或页面内成功/失败提示是操作反馈，不算通知渠道，不进入通知中心。
