# 多来源客户端行为结论

本文记录多来源显示、保存位置和通知偏好的公开讨论结论。长期迁移边界见 ADR 0019；后端内部接口、数据模型和实现切片不在本文展开。

## 来源显示

- 客户端来源平级，不设主账号/次账号。
- 本机模式也允许连接多个服务端和公开来源。
- 主导航按业务模块组织，不按来源或空间拆分。
- 列表默认合并展示所有可用来源，来源和空间只做轻量标签、筛选和状态信息。
- 顶部操作区顺序暂定为主题切换、通知、来源状态、头像。
- 来源状态入口使用图标加状态点，不做单独浮层；正常时弱显示，有来源离线、登录失效、迁出、待同步或同步失败时叠加黄点/红点。
- 点击来源状态图标直接进入 `设置 -> 数据与来源`。
- `数据与来源` 页面展示来源状态摘要、来源列表、默认保存位置、迁移入口和具体来源操作。

## 来源不可用

- 某个来源不可用时，不打断整个页面，其他来源继续展示和操作。
- 已缓存内容可以只读展示，并标记可能不是最新。
- 不可用来源不允许写操作，例如完成、编辑、删除、加入日程或修改通知设置。
- 当前筛选只看不可用来源时，页面显示该来源不可用空态；如果没有缓存，则显示无法加载。
- 某个业务来源登录失效只影响该来源，不把整个客户端踢出主界面。
- Web/Desktop Online 如果没有任何可用业务会话，不进入主界面，停在登录页或业务入口选择页。
- 匿名公开来源不可用时轻处理，公开流少显示对应内容即可，不做大面积报错。

## 默认保存位置

- 客户端有一个全局 `默认保存位置`。
- 能进入主界面时，至少存在一个当前可写的保存位置。
- 普通新增、首页快捷新增、公开事项加入个人 Todo / 日程，默认保存到该位置。
- 默认保存位置只能选择当前可写的业务来源或空间；只读公开来源不能作为保存目标。
- 选择来源时可以勾选 `设为默认保存位置`；不勾选只对本次有效。
- 默认保存位置不可写、离线或权限失效时，不自动改默认值；新增时要求用户重新选择保存位置。
- 空间上下文和规则生成内容不使用全局默认保存位置。
- 不设计“没有可写来源”的新增空态；如果在线端没有任何可用业务会话，用户会先回到登录页或业务入口选择页。

## 设置页入口

- 头像菜单只放 `设置` 和 `退出登录`。
- 在线或服务端登录态显示 `退出登录`；本机固定身份不能手动登录，不显示退出。
- 如果本机模式同时连接服务端，退出某个服务端来源的操作放在 `设置 -> 数据与来源` 的具体来源项里，不放头像全局菜单。
- 当前端存在待同步操作时，例如移动端 App 离线暂存队列未清空，退出登录或退出某个来源需要拦截，提示先同步或放弃。
- 设置页分组暂定为个人、通知、数据与来源、外观、安全。
- `实例管理` 只对实例管理员显示；空间管理员不等于实例管理员，不能看到实例管理。
- Desktop Full 本机固定管理员可以视为本机实例管理员，但页面文案应偏“本机管理”，避免误解为远端服务器管理。

## 在线端退出与登录页

- 在线端退出登录后清除当前服务端账号会话和本地敏感登录态，回到未登录状态。
- 退出某个业务来源后，如果仍有其他可用业务会话，继续留在主界面，只移除该来源的账号内容。
- 退出后如果没有任何可用业务会话，Web/Desktop Online 回到登录页或业务入口选择页。
- Desktop Full 不因为远端业务来源退出或失效离开本机首页。
- 不清除主题、语言、窗口偏好、已保存业务入口地址、公开只读来源和客户端布局设置。
- 不删除缓存，但退出后不展示需要该账号权限的缓存业务内容。
- Web Online 退出后清除 Web session cookie 和 CSRF 相关登录态。
- Desktop Online 退出后清除 Rust 主进程内 access token 和 refresh token，保留远端地址配置。
- 登录页只处理业务入口，不显示公开入口。
- 登录页展示已添加业务入口列表，包括名称、地址、状态和登录态；用户选择业务入口后，再显示该入口支持的登录方式。
- 登录页可以添加新的业务入口：输入地址，连接检查通过后保存并进入登录。
- Desktop Full 启动时可以直接进入本机数据首页；添加业务入口或公开入口在 `设置 -> 数据与来源` 中完成。

## 业务入口与公开入口

- 业务入口和公开入口分开维护，不要求属于同一个服务端实例，不校验同一个 `instanceId`，不自动合并。
- 业务入口用于登录、Todo、日程、通知、规则、设置和迁移等需要账号或业务权限的能力。
- 公开入口用于公开发现、公开流、公开主页和公开事项预览等匿名只读能力。
- 登录页只显示和管理业务入口。
- 公开入口登录后在 `设置 -> 数据与来源` 里添加、编辑、移除。
- 如果只有公开入口，就是匿名只读公开来源；如果只有业务入口，也可以登录使用，但不提供公开发现。
- 来源列表中业务入口来源和公开入口来源可以分别显示；即使它们实际来自同一个部署，系统也不把它们当成同一个来源。
- 从公开事项加入个人 Todo / 日程时，写入默认保存位置，不要求默认保存位置和公开入口来自同一个服务端。
- 已添加过的同地址入口不重复添加，提示用户去已有来源登录或管理。
- 入口地址不建议在原连接上直接编辑；需要变更地址时移除后重新添加，普通编辑只允许改显示名称。

## 公开入口归属

- 公开入口按归属栏管理，不做自动同步。
- 归属栏包括每个已登录业务账号，以及最下面的 `本机保存` 栏。
- 新增公开入口时必须选择保存到哪个归属栏：账号 A、账号 B 或本机保存。
- 每条公开入口同一时间只属于一个归属栏；移动公开入口就是改变归属。
- 账号栏里的公开入口跟随该业务账号同步；本机保存栏里的公开入口只保存在当前客户端。
- 多个业务账号各自拥有自己的公开入口列表，不跨账号自动合并。
- 同一个归属栏里不允许重复地址；跨归属栏允许同地址入口，因为它们属于不同账号或本机配置。
- 公开入口地址不建议在原连接上直接编辑；需要变更地址时移除后重新添加。
- 普通编辑只允许修改显示名称、启用状态和排序等连接配置。
- 公开内容缓存仍是本地缓存，可随时丢弃，不跟随公开入口归属同步。

## 公开入口操作

- 账号栏里的入口提供 `转为本机` 和 `删除`。
- `转为本机` 是把入口从当前账号栏移动到本机保存栏，不再跟随该账号同步。
- 本机保存栏里的入口提供 `同步到账号` 和 `删除`。
- `同步到账号` 是把入口从本机保存栏移动到用户选择的账号栏。
- 删除只删除当前归属栏里的记录，不自动删除其他归属栏里的同地址入口。
- 如果移动目标栏里已经有同地址入口，不新增重复项，只移除原归属栏里的入口，并提示目标栏已存在。
- 批量操作第一版支持批量转为本机、批量同步到账号和批量删除。
- 批量启用、批量停用可以作为增强操作；如果实现，仍只影响当前选择的归属栏记录。

## 通知中心与来源

- 通知中心默认合并展示所有来源的站内通知。
- 每条通知带轻量来源标识，并支持按来源筛选。
- 未读角标显示所有来源的未读总数。
- 离线来源只显示本地已有缓存通知，不拉取新通知。
- 点击通知时，来源在线且内容可访问则打开对应内容；来源离线则提示暂时不可用；来源已迁出/已注销则只展示通知文本。
- 强制系统通知可以进入合并收件箱，并带来源标识。

## 通知偏好

- 客户端有一套全局通知偏好，作为所有来源的默认值。
- 新增服务端来源时默认跟随客户端全局偏好。
- 每个来源可以选择跟随客户端默认或自定义此来源。
- 具体事项仍支持 `default` / `custom`：`default` 跟随该来源当前有效默认值，`custom` 使用事项自己的通知设置。
- 客户端设备静音是设备级设置，覆盖所有来源的声音、桌面通知和横幅，不影响站内收件箱。
- 系统关键通知最多只能强制进入站内收件箱，不能强制声音、桌面通知或弹出式横幅。
- Toast 或页面内成功/失败提示是操作反馈，不算通知渠道，不进入通知中心。
