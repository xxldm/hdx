# 多来源客户端行为结论

本文记录多来源显示、保存位置和通知偏好的公开讨论结论。长期迁移边界见 ADR 0019；后端内部接口、数据模型和实现切片不在本文展开。

## 来源显示

- 客户端来源平级，不设主账号/次账号。
- 本机模式也允许连接多个服务端和公开来源。
- 主导航按业务模块组织，不按来源或空间拆分。
- 列表默认合并展示所有可用来源，来源和空间只做轻量标签、筛选和状态信息。
- 顶部操作区顺序暂定为主题切换、通知、来源状态、头像。
- 来源状态入口使用图标加状态点，不做单独浮层；正常时弱显示，有来源离线、登录失效、待同步或同步失败时叠加黄点/红点。
- 点击来源状态图标直接进入 `设置 -> 数据与来源`。
- `数据与来源` 页面展示来源状态摘要、业务来源、公开入口、默认保存位置和具体来源操作。
- 本机数据只在 Desktop Full 中作为一个业务来源出现；本机存储、健康和迁移入口都放在本机数据或对应服务端来源详情里，不单独做一级区块。

## 来源状态

- 来源主状态包括 `可用`、`只读`、`受限`、`需要登录`、`不可用`、`已迁出`。
- `可用` 表示来源在线、登录态有效，并且当前用户可读写。
- `只读` 表示来源可以查看但不能写入，例如公开只读来源、权限变化后的只读来源或移动端离线只读内容。
- `受限` 表示来源在线且登录有效，但因为配额、迁移频率或实例策略限制了部分写操作。
- `需要登录` 表示业务入口仍保存，但当前没有有效登录态。
- `不可用` 表示连接失败、服务端异常或来源暂时无法访问。
- `已迁出` 表示服务端账号已迁出到本机，只保留连接记录，不参与聚合，不再作为保存目标。
- `已移除`、`已注销` 只在历史通知或历史引用里展示，不作为正常来源列表里的主状态。
- 辅助标记不升级为主状态，包括 `默认保存位置`、`有待同步`、`同步失败`、`登录即将过期`、`连接慢`、`只读公开入口`、`本机数据`。
- 状态点颜色：绿色表示可用；蓝色表示只读或本机数据；黄色表示需要处理但不紧急；红色表示不可用、同步失败或不可继续使用；灰色表示停用、已迁出或历史状态。
- 来源列表排序：默认保存位置靠前，本机数据固定靠前，可用业务来源靠前，需要登录、受限和不可用来源排在后面但不隐藏。
- 公开入口按归属栏展示，不混进业务来源排序。
- 顶部来源状态摘要只显示异常和关键状态；正常时弱显示或只显示 `所有来源可用`。
- 已迁出是用户主动发起后的历史状态，不触发右上角黄点或红点，也不进入顶部异常摘要。
- 点击右上角来源状态图标直接进入 `设置 -> 数据与来源`，不在顶部图标上做复杂浮层。

## 来源不可用

- 某个来源不可用时，不打断整个页面，其他来源继续展示和操作。
- Web/Desktop Online 按普通在线页面处理，不承诺来源不可用时展示历史缓存；接口不可用时显示不可用空态。
- 移动端 App 在离线暂存范围内可以只读展示本设备已有内容，并标记可能不是最新。
- 不可用来源不允许写操作，例如完成、编辑、删除、加入日程或修改通知设置。
- 当前筛选只看不可用来源时，页面显示该来源不可用空态；移动端如果没有本地可用内容，则显示无法加载。
- 某个业务来源登录失效只影响该来源，不把整个客户端踢出主界面。
- Web/Desktop Online 如果没有任何可用业务会话，不进入主界面，停在登录页或业务入口选择页。
- 匿名公开来源不可用时轻处理，公开流少显示对应内容即可，不做大面积报错。
- 右上角来源状态点只提示有来源需要处理，点击进入 `设置 -> 数据与来源`。
- 业务列表里部分来源不可用时，顶部显示轻提示：`部分来源暂时不可用，列表可能不完整。`
- 部分来源不可用轻提示可以关闭；关闭只在本次页面会话内有效，刷新或来源状态变化后可以再次出现。
- 轻提示可以提供 `查看来源` 操作，跳转到 `设置 -> 数据与来源`。
- 当前筛选只看需要登录来源时，空态文案为：`这个来源需要重新登录后才能查看。`
- 当前筛选只看不可用来源时，空态文案为：`暂时无法连接这个来源。`
- 当前筛选只看受限来源时，空态或提示文案为：`这个来源部分操作受限。`
- 当前筛选只看只读来源时，空态或提示文案为：`当前只能查看，不能修改。`
- 操作按钮因为来源状态不可用而不能执行时，只给原因提示，不弹确认弹窗。
- 操作提示示例包括：`来源需要登录，暂时不能修改。`、`来源暂时不可用，稍后再试。`、`当前只有查看权限。`
- 移动端离线暂存范围内显示本设备已有内容时，提示：`当前离线，正在显示本设备已有内容。`

## 多来源列表与分页

- Todo、日程、通知等业务列表默认合并展示当前可访问来源，不按服务端拆一级页面。
- 每条内容只显示轻量来源标识，例如 `本机`、`个人区`、`空间 / 家庭`，不在列表里长期铺完整地址或服务端层级。
- 来源和空间标签使用中性样式，不按来源生成大量彩色标签；可以用小图标或状态点辅助识别。
- 当前筛选已经限定到某个来源时，可以弱化或隐藏来源标签；列表只来自一个来源时可以不显示来源标签。
- 空间标签只在空间内容中显示；个人区标签只在个人区和空间混排时显示，全是个人区时不显示。
- 列表顶部提供来源、空间、状态等筛选；来源筛选支持多选，默认不筛。
- 默认保存位置不影响列表筛选；保存到哪里和当前查看哪些内容是两件事。
- 点击列表项上的来源标签时，第一版筛选到该来源，不打开来源详情。
- 点击列表项上的空间标签时，第一版筛选到该空间，不打开空间或来源详情。
- 管理来源统一从右上角来源状态入口进入 `设置 -> 数据与来源`。
- 业务列表使用游标分页，不使用页码分页；UI 表现为 `加载更多` 或连续加载。
- 多来源聚合分页由端侧 BFF 或数据访问层维护聚合游标，页面层只消费统一列表和不透明 `nextCursor`。
- Web 的聚合放在 Nuxt server BFF；Desktop Online 和 Desktop Full 放在 Rust BFF、sidecar 或等价端侧数据访问层；移动端放在 App 数据仓库层。
- 端侧聚合不代表远端服务端之间互信或互相拉取数据。
- 聚合游标需要包含各来源自己的游标和当前排序上下文；UI 不解析游标内容。
- 聚合分页只保证继续加载下一批，不保证稳定第 N 页。
- 列表刷新时回到顶部重新拉取；加载更多时使用上一次返回的聚合游标。
- 某个来源不可用时，该来源不参与新一批加载；页面提示部分来源暂时不可用，列表可能不完整。
- 已经加载在当前页面里的内容可以继续显示，但来源不可用时需要用轻量状态标记可能不是最新。
- 新增、完成、删除等操作后可以局部更新当前列表；下一次刷新以来源事实结果为准。

## 筛选状态

- Todo、日程、通知等模块各自维护筛选状态，不做全局联动。
- 筛选状态默认只保存在当前设备，不跨设备同步。
- 首页组件和完整页面的筛选状态分开维护；完整页面筛选不反向影响首页组件。
- Web 完整页面筛选可以进入 URL query，方便刷新、返回和分享当前视图。
- URL query 只放非敏感筛选键，不放 token、地址或其他敏感值。
- 首页组件筛选不进入 URL。
- 筛选条件失效时，例如来源被移除或空间权限消失，自动移除失效条件并给轻提示。
- 默认保存位置不受筛选影响；来源筛选只决定当前看哪些内容，不决定新增保存到哪里。

## 多来源排序稳定性

- 多来源列表排序字段必须来自统一列表投影，不由页面层临时猜测。
- 列表项需要携带 `sortKey` 或等价稳定排序字段；同一排序位置用来源 ID 和内容 ID 兜底打散。
- 日程列表按开始时间升序；进行中事项自然靠前或处在前段。开始时间相同，再按来源显示名和事项 ID 稳定排序。
- Todo 列表先按无时间、逾期、今天、未来、已完成等分组，再按 Todo 模块自己的组内规则排序。
- 无时间 Todo 组置顶且可折叠；组内优先使用用户手动排序，没有手动排序时按创建时间倒序。
- 列表展示按当前设备或账号偏好的时区显示；排序使用来源返回的规范时间值，不使用格式化后的显示文本。
- 来源不可用时，新一批加载不包含该来源；已加载内容保留当前位置，不为了补齐排序大幅跳动。

## 合并列表写操作

- 合并业务列表中的单条内容操作永远回到内容所属来源执行。
- 默认保存位置只影响新增，不影响已有内容的编辑、完成、删除、通知设置或其他操作归属。
- 只读、需要登录或不可用来源里的内容不能直接写入；如果内容仍可见，操作按钮禁用或点击后提示原因。
- 已迁出来源不参与业务列表聚合，不展示可操作业务内容。
- 不支持跨来源移动、复制、同步或智能合并。
- Todo 第一版不做批量完成、批量取消完成或批量删除。
- 删除、取消、迁移、公开、取消公开等高风险操作不做跨来源批量操作。
- 通知中心可以保留批量标记已读；执行时按来源分组处理，部分来源失败时展示摘要。
- 批量结果文案需要说明成功数量和失败原因，例如 `已标记 8 条，2 条因来源需要登录未处理`。

## 缓存与离线暂存

- 第一版不做通用用户可见缓存管理，不提供 `清理缓存` 按钮。
- Web/Desktop Online 是在线页面；浏览器或 WebView 的临时缓存按平台机制处理，产品不承诺离线可用。
- Desktop Full 的本机数据是真实数据，不叫缓存；远端服务端内容仍按在线来源处理。
- 移动端 App 才设计本地数据库缓存、白名单离线暂存和待同步队列；细节见移动端离线暂存文档。
- 移除连接、切换账号或退出登录时清理当前设备相关本地状态，属于流程内部收尾，不作为独立缓存管理功能。

## 默认保存位置

- 客户端有一个全局 `默认保存位置`。
- 能进入主界面时，至少存在一个当前可写的保存位置。
- 普通新增、首页快捷新增、公开事项加入个人 Todo / 日程，默认保存到该位置。
- 默认保存位置可以选择本机数据、某个服务端账号下的个人区，或某个空间。
- 默认保存位置需要按保存边界分开显示，例如 `本机数据`、`个人区`、`空间 / 家庭`。
- 默认保存位置不能选择公开入口、只读来源、已迁出来源、登录失效来源，或当前用户没有创建权限的空间。
- 选择来源时可以勾选 `设为默认保存位置`；不勾选只对本次有效。
- 默认保存位置不可写、离线、登录失效、权限失效或被移除时，不自动改成其他位置；新增时用 `UModal` 要求用户重新选择保存位置，并可以勾选 `设为默认保存位置`。
- 切换账号或移除连接导致默认保存位置消失时，清除默认绑定；下次新增时再选，不自动猜测。
- 空间上下文新增默认写入当前空间，不使用全局默认保存位置。
- 规则生成内容不使用全局默认保存位置，写入规则所在来源。
- 不设计“没有可写来源”的新增空态；如果在线端没有任何可用业务会话，用户会先回到登录页或业务入口选择页。

## 新增入口归属

- 首页快捷新增默认使用全局默认保存位置。
- 首页快捷新增的基础表单只展示标题和时间；展开后填写描述、地点、链接和通知设置。
- 完整页面新增默认使用全局默认保存位置。
- 普通列表筛选不改变新增保存位置；即使当前筛选只看某个来源或空间，新增仍按默认保存位置处理。
- 真正处于空间上下文页面时，新增默认写入当前空间。
- 从公开事项加入个人 Todo / 日程时默认使用全局默认保存位置；如果第一次加入需要选择保存位置，可以勾选记住默认。
- 公开事项加入个人安排的默认保存位置需要能在设置里修改。
- 规则生成写入规则所在来源，不使用全局默认保存位置。
- 默认保存位置失效时，新增用 `UModal` 选择保存位置，并可勾选 `设为默认保存位置`。

## 设置页入口

- 头像菜单只放 `设置` 和 `退出登录`。
- 在线或服务端登录态显示 `退出登录`；本机固定身份不能手动登录，不显示退出。
- 如果本机模式同时连接服务端，退出某个服务端来源的操作放在 `设置 -> 数据与来源` 的具体来源项里，不放头像全局菜单。
- 当前端存在待同步操作时，例如移动端 App 离线暂存队列未清空，退出登录或退出某个来源需要拦截，提示先同步或放弃。
- 设置页分组暂定为个人、通知、数据与来源、外观、安全。
- `实例管理` 只对实例管理员显示；空间管理员不等于实例管理员，不能看到实例管理。
- Desktop Full 本机固定管理员可以视为本机实例管理员，但页面文案应偏“本机管理”，避免误解为远端服务器管理。

## 业务来源详情

- 业务来源详情按区块展示，不把所有操作堆在一个长列表里。
- 基本信息展示名称、地址、来源类型、当前状态、当前登录账号和最近连接时间。
- 来源类型包括本机数据、服务端账号和空间。
- 当前登录账号展示公开名称、邮箱和账号状态；本机数据展示固定本机身份。
- 能力与数据范围展示 Todo、日程、通知、规则、空间等模块是否启用或可访问。
- 服务端来源在能力与数据范围区展示配额摘要；本机数据展示本机存储与健康。
- 默认保存位置区展示当前来源或空间是否为默认保存位置，并提供设为默认操作。
- 通知偏好区展示跟随全局或自定义此来源。
- 常规操作包括检查连接、重新登录、退出登录、修改显示名称和打开通知偏好。
- 退出登录属于常规操作；它不删除服务端数据，也不是不可逆操作，但有待同步内容时需要拦截。
- 危险操作放在详情底部，包括移除连接和迁移。
- 删除公开入口属于公开入口详情，不放在业务来源详情。
- 已迁出的服务端账号不能重新登录、不能换账号，只能移除连接记录。
- Desktop Full 本机数据不能退出登录，不能移除，只能修改显示名称和查看本机管理入口。
- 没有可写权限、处于只读或已迁出状态的来源不能设为默认保存位置。
- 当前端存在待同步操作时，例如移动端 App 离线暂存队列未清空，退出登录、退出某个来源或移除连接需要拦截。

## 重新登录与切换账号

- 普通服务端来源重新登录后，需要校验登录成功的账号是否和当前来源原账号一致。
- 如果是同一个账号，直接恢复会话，保留原来源设置、默认保存位置和通知偏好。
- 如果不是同一个账号，视为切换账号，需要让用户确认是否继续。
- 用户确认切换后，客户端清除当前连接在本机保存的旧账号数据、登录态、本地状态、来源偏好、默认保存位置绑定和来源级通知偏好，再把该来源接入为新账号。
- 用户取消切换时，丢弃这次新登录态，回到重新登录流程。
- 切换账号不会删除旧账号在服务端的数据。
- 当前来源存在待同步操作时，不能直接切换账号；需要先同步完成，或由用户明确放弃待同步内容。
- 已迁出的服务端账号不适用切换账号流程，仍然只能移除此连接记录。

## 移除连接记录

- 业务来源连接记录第一版不跨设备同步；移除业务来源只影响当前设备。
- 移除业务来源使用 `UModal` 确认，不使用 `UPopover`。
- 移除只清除当前客户端里的连接记录、登录态、本地状态、来源显示名、来源级通知偏好、默认保存位置绑定和同步临时状态。
- 移除不会删除服务端账号，也不会删除服务端上的 Todo、日程、规则、设置或其他业务数据。
- 当前设备需要同时清理该来源的列表状态、详情状态、通知本地状态、来源级索引和临时状态；移动端还需要清理该来源的离线暂存残留。
- 如果该来源是默认保存位置，确认时提示移除后需要重新选择默认保存位置。
- 移除完成后，如果还有其他可写来源，引导用户选择新的默认保存位置。
- Web/Desktop Online 移除后如果没有任何可用业务会话，回到登录页或业务入口选择页。
- Desktop Full 移除远端业务来源后仍留在本机数据首页。
- 当前来源存在待同步操作时，不能静默移除；需要先同步完成，或由用户明确选择放弃待同步内容并移除。
- 移除业务来源不会自动删除或转移账号栏里的公开入口；这些公开入口仍属于对应账号。用户如果想保留到当前设备本机，需要移除前先转为本机保存。
- 已迁出的服务端账号移除时，只清掉历史连接记录和本地残留状态，不影响已经迁到本机的数据。

## 配额与限制

- 服务端业务来源详情需要展示 `配额与限制`。
- 迁入不做允许或禁止开关；本机迁入服务端默认是可用能力，实际能否完成只看账号状态、项目配额和迁移频率。
- 配额优先按项目数量展示，不优先按字节大小展示。
- 常见数量配额包括 Todo 数量、日程数量、规则数量、候选项数量、标签数量、附件数量、公开事项数量和公开主页数量。
- 公开事项和公开主页配额只在账号具备公开能力时展示。
- 字节容量只用于附件、本地上传声音、文件等真实文件资源；没有文件能力时不显示容量配额。
- 来源详情默认只显示配额摘要，例如 `Todo 120/1000 · 日程 45/500 · 规则 8/50`，展开后再显示完整限制。
- 迁移前按要迁入的项目数量预估；如果某类项目会超限，阻止迁移并明确说明哪类超限。
- 迁移频率作为限制展示，例如每 7 天 1 次；达到限制时显示下次可迁移时间。
- 接近配额上限时来源状态可以显示黄色提示；超出配额或触发频率限制时，来源可进入 `受限` 状态。

## 本机存储与健康

- Desktop Full 本机数据不使用服务端治理型限制。
- 本机数据详情不显示 `配额与限制`，改显示 `本机存储与健康`。
- 本机数据不限制 Todo、日程、规则、候选项、标签或公开内容的项目数量。
- 本机数据不使用迁移频率限制、公开发布配额、账号级策略限制、空间成员限制、实例治理限制或服务端反滥用限制。
- 本机数据只受真实本机资源和系统状态影响，例如磁盘空间、文件路径权限、数据库健康、附件目录可访问性、本机 sidecar 可用性和操作系统权限。
- 本机 -> 服务端迁移时，检查目标服务端的项目配额、文件资源配额和迁移频率。
- 服务端 -> 本机迁移时，只检查本机资源是否能承载，不检查项目数量配额或迁移频率。
- 如果后续需要清理本机空间，应作为存储管理能力，不叫配额。

## 迁移入口与确认

- 本机数据迁移只在 Desktop Full 中出现。
- Web/Desktop Online 没有长期本机数据容器，不显示本机 -> 服务端 或 服务端 -> 本机迁移入口。
- 移动端 App 第一版不显示这套迁移入口；移动端本地数据库只用于白名单离线暂存，不作为长期本机数据模式。
- 迁移入口放在 `设置 -> 数据与来源 -> 业务来源详情 -> 迁移`。
- 本机数据详情显示 `迁移到服务端`；服务端业务来源详情显示 `迁出到本机`。
- 已迁出的服务端账号只显示状态和 `移除此连接记录`，不再显示迁移入口。
- 迁移确认使用 `UModal`，不使用 `UPopover`；迁移需要展示检查结果、后果确认和执行状态。
- 迁移 `UModal` 按区块展示来源/目标、预计项目数量、检查结果、后果确认和主操作按钮。
- 检查未通过时禁用主操作按钮，并明确说明失败原因；失败后不清理任何数据，允许用户处理后重试。
- 迁移执行中不允许直接关闭弹窗；如果后续允许关闭，需要二次确认并说明关闭不等于取消后台任务。

本机 -> 服务端：

- 弹窗展示目标服务端和目标账号。
- 预计项目数量包括 Todo、日程、规则、候选池、候选项、标签、通知偏好、本地上传声音等。
- 检查目标服务端的项目配额、文件资源配额和迁移频率。
- 成功后清理本机业务数据，用户之后在目标服务端继续使用。
- 不清理公开入口、业务入口地址、登录态、客户端布局设置和其他纯客户端配置。
- 主按钮文案为 `确认迁移到服务端`。
- 成功后目标服务端必须生成一条站内通知进入收件箱；不发声音、桌面通知或强制横幅，Toast 只作为操作反馈。

服务端 -> 本机：

- 弹窗展示来源服务端和来源账号。
- 预计项目数量包括即将迁出的个人 Todo、日程、规则、候选池、候选项、标签、通知偏好和文件资源。
- 只检查本机资源是否能承载，不检查项目数量配额或迁移频率。
- 明确提示每个服务端账号只能迁出到本机一次。
- 明确提示迁出成功后服务端账号不可继续使用，客户端连接状态会变成 `已迁出`，之后只能移除此连接记录。
- 明确提示只删除已迁出的个人数据；未删除的数据里该用户显示为 `已注销用户`，系统保留邮箱等基础账号信息用于禁止重复注册和安全审计。
- 主按钮文案为 `确认迁出到本机`。
- 成功后本机收件箱必须生成一条站内通知；服务端账号进入不可恢复的 `已迁出` 状态。
- 没有公开权限、没有发布公开内容的账号，不显示公开事项相关警告。

## 在线端退出与登录页

- 在线端退出登录后清除当前服务端账号会话和本地敏感登录态，回到未登录状态。
- 退出某个业务来源后，如果仍有其他可用业务会话，继续留在主界面，只移除该来源的账号内容。
- 退出后如果没有任何可用业务会话，Web/Desktop Online 回到登录页或业务入口选择页。
- Desktop Full 不因为远端业务来源退出或失效离开本机首页。
- 不清除主题、语言、窗口偏好、已保存业务入口地址、公开只读来源和客户端布局设置。
- 退出后不展示需要该账号权限的内容。
- Web Online 退出后清除 Web session cookie 和 CSRF 相关登录态。
- Desktop Online 退出后清除 Rust 主进程内 access token 和 refresh token，保留远端地址配置。
- 登录页只处理业务入口，不显示公开入口。
- 登录页展示已添加业务入口列表，包括名称、地址、状态和登录态；用户选择业务入口后，再显示该入口支持的登录方式。
- 登录页可以添加新的业务入口：输入地址，连接检查通过后保存并进入登录。
- Desktop Full 启动时可以直接进入本机数据首页；添加业务入口或公开入口在 `设置 -> 数据与来源` 中完成。

## 添加来源

- 业务入口可以在登录页和 `设置 -> 数据与来源` 添加。
- 添加业务入口时输入业务入口地址，连接检查通过后展示服务端名称、登录方式和能力摘要，再进入登录。
- 同一个服务端在同一个客户端里只允许接入一个账号。
- 业务入口地址规范化后命中已有来源时，不新增第二个业务来源，提示用户去已有来源登录或管理。
- 如果用户对已有服务端登录成另一个账号，按重新登录与切换账号流程处理；确认切换后替换当前来源账号，不并存两个账号。
- 公开入口只在 `设置 -> 数据与来源 -> 公开入口` 添加。
- 添加公开入口时必须选择归属栏：某个账号或本机保存。
- 添加公开入口检查通过后展示公开发现、公开流、公开主页和公开事项预览等能力摘要。
- 业务入口和公开入口不互相推导；添加业务入口不会自动添加公开入口，添加公开入口也不会自动添加业务入口。
- 同一个归属栏里的公开入口地址规范化后不允许重复；跨归属栏允许同地址入口。

## 业务入口与公开入口

- 业务入口和公开入口分开维护，不要求属于同一个服务端实例，不校验同一个 `instanceId`，不自动合并。
- 业务入口用于登录、Todo、日程、通知、规则、设置和迁移等需要账号或业务权限的能力。
- 公开入口用于公开发现、公开流、公开主页和公开事项预览等匿名只读能力。
- 登录页只显示和管理业务入口。
- 公开入口登录后在 `设置 -> 数据与来源` 里添加、编辑、移除。
- 如果只有公开入口，就是匿名只读公开来源；如果只有业务入口，也可以登录使用，但不提供公开发现。
- 来源列表中业务入口来源和公开入口来源可以分别显示；即使它们实际来自同一个部署，系统也不把它们当成同一个来源。
- 从公开事项加入个人 Todo / 日程时，写入默认保存位置，不要求默认保存位置和公开入口来自同一个服务端。
- 入口地址不建议在原连接上直接编辑；需要变更地址时移除后重新添加，普通编辑只允许改显示名称。

## 公开入口归属

- 公开入口按归属栏管理，不做跨归属栏自动合并或自动复制。
- 归属栏包括当前设备上每个已登录业务账号，以及最下面的 `本机保存` 栏。
- 新增公开入口时必须选择保存到哪个归属栏：某个已登录业务账号或本机保存。
- 每条公开入口同一时间只属于一个归属栏；移动公开入口就是改变归属。
- 账号栏里的公开入口跟随该业务账号同步；本机保存栏里的公开入口只保存在当前客户端。
- 多个业务账号各自拥有自己的公开入口列表，不跨账号自动合并。
- 同一个归属栏里不允许重复地址；跨归属栏允许同地址入口，因为它们属于不同账号或本机配置。
- 公开入口地址不建议在原连接上直接编辑；需要变更地址时移除后重新添加。
- 普通编辑只允许修改显示名称、启用状态和排序等连接配置。
- 公开内容的临时页面数据仍属于当前设备，可随时丢弃，不跟随公开入口归属同步；移动端离线缓存另按移动端规则处理。

## 公开入口操作

- 账号栏里的入口提供 `转为本机` 和 `删除`。
- `转为本机` 是把入口从当前账号栏移动到本机保存栏，不再跟随该账号同步。
- 本机保存栏里的入口提供 `同步到账号` 和 `删除`。
- `同步到账号` 是把入口从本机保存栏移动到用户选择的账号栏。
- 删除只删除当前归属栏里的记录，不自动删除其他归属栏里的同地址入口。
- 如果移动目标栏里已经有同地址入口，不新增重复项，只移除原归属栏里的入口，并提示目标栏已存在。
- 批量操作第一版支持批量转为本机、批量同步到账号和批量删除。
- 批量启用、批量停用可以作为增强操作；如果实现，仍只影响当前选择的归属栏记录。

## 公开入口详情

- 公开入口详情展示显示名、公开入口地址、所属归属栏、状态、最近检查时间和能力摘要。
- 所属归属栏包括某个已登录业务账号或本机保存。
- 状态包括可用、不可用和停用。
- 能力摘要包括公开发现、公开流、公开主页和公开事项预览。
- 常规操作包括检查连接、修改显示名、启用、停用、转为本机、同步到账号、删除和打开公开发现。
- 公开入口不能作为默认保存位置。
- 公开入口地址不直接编辑；需要变更地址时删除后新增。
- 账号栏里的公开入口属于该账号偏好；本机保存栏里的公开入口只存在当前客户端。
- 删除只删除当前归属栏记录，不跨栏删除其他同地址入口。

## 通知中心与来源

- 通知中心默认合并展示所有来源的站内通知。
- 每条通知带轻量来源标识，并支持按来源筛选。
- 未读角标显示所有来源的未读总数。
- 移动端离线暂存范围内，离线来源可以显示本地已有通知内容；Web/Desktop Online 不承诺离线通知缓存。
- 点击通知时，来源在线且内容可访问则打开对应内容；来源离线则提示暂时不可用；来源已迁出/已注销则只展示通知文本。
- 强制系统通知可以进入合并收件箱，并带来源标识。
- 公开入口本身不产生通知中心通知；公开流更新不按通知处理。
- 用户把公开事项加入个人 Todo / 日程后，后续提醒按保存位置所属来源进入通知中心。

## 通知偏好

- 客户端有一套全局通知偏好，作为所有来源的默认值。
- 新增服务端来源时默认跟随客户端全局偏好。
- 每个来源可以选择跟随客户端默认或自定义此来源。
- 具体事项仍支持 `default` / `custom`：`default` 跟随该来源当前有效默认值，`custom` 使用事项自己的通知设置。
- 客户端设备静音是设备级设置，覆盖所有来源的声音、桌面通知和横幅，不影响站内收件箱。
- 系统关键通知最多只能强制进入站内收件箱，不能强制声音、桌面通知或弹出式横幅。
- Toast 或页面内成功/失败提示是操作反馈，不算通知渠道，不进入通知中心。
