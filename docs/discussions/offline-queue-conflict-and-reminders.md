# 移动端离线暂存、冲突处理与本地提醒结论

本文记录移动端 App 弱网/无网暂存、冲突处理、写操作幂等和本地提醒兜底的公开讨论结论。
ADR 0016 仍是用户数据持久化与同步边界的长期事实源；本文补充尚未进入接口草案和实现计划的细节。

## 离线暂存边界

- 移动端 App 离线能力采用白名单；只有客户端能本地完成校验、保存和展示结果的功能才允许离线操作。
- 第一版白名单以个人 Todo / 日程、追加备注、标记通知已读、稍后提醒等本地可记录操作为主；具体模块接入时单独声明。
- 规则生成是后台行为，客户端离线时不本地执行规则生成。
- 规则、候选池、公开发布、协作、空间权限、账号安全和迁移等依赖后端编排或权限判断的功能，离线时不可编辑。
- 非白名单功能不进入离线队列，也不产生离线冲突处理。
- 服务端通知计划由后端/通知中心维护；客户端离线时不伪造远端通知计划。
- 离线新建或修改带提醒的事项时，客户端可以生成本机临时提醒；同步成功后由服务端正式接管。
- 站内收件箱以后端为准，本机临时提醒不等于服务端已投递通知。
- 依赖联网的功能离线时不能只置灰或消失；必须醒目提示需要联网后使用。
- 已缓存但离线不可编辑的内容可以只读展示，并标记 `离线只读`。
- 用户点击不可用操作时，提示示例：`当前离线，规则编辑需要联网后使用。`

## 移动端本地缓存

- 移动端 App 为弱网/无网暂存保留离线可用缓存，支持长时间离线继续处理本机已有和本机新增内容，但不承诺拥有完整远端视图。
- 移动端 App 缓存不要仅因为技术过期就让仍按业务规则可见的内容消失；可以标记很久未同步、限制部分操作或等待重连校正。
- 缓存内容展示必须继续遵守业务结束、取消、删除、隐藏和保留窗口规则。
- 离线时用客户端已知数据和本机时间计算可见性；联网后用服务端最新状态修正。
- 缓存清理只影响本机还能不能离线打开内容，不单独决定默认列表可见性。
- 没缓存过的数据离线时不可见，例如未打开过的历史内容、未同步下来的空间或离线期间新增的远端公开内容。

## 写操作幂等

- 除查询外，所有进入服务端的写请求都带 `clientOperationId`；在线页面提交和移动端离线队列重放都遵守同一要求。
- `clientOperationId` 由客户端生成，建议 UUID 或 ULID。
- 同一个 UI 提交流程内，多次点击、按钮连点和网络重试复用同一个 `clientOperationId`。
- 提交中按钮应进入 loading 或 disabled，前端先防连点。
- 用户完成一次流程后重新打开表单、重新编辑并再次保存，才生成新的 `clientOperationId`。
- 同一个 `clientOperationId` 重复提交但 payload 不一致时，服务端应拒绝并返回稳定错误码。
- 创建类操作成功后，服务端需要返回客户端临时 ID 与正式 ID 的映射。

适合幂等命令的操作包括：

- 完成或取消完成 Todo。
- 追加备注。
- 新增子项。
- 稍后提醒。
- 标记通知已读。
- 从公开事项加入个人日程或 Todo。

## 移动端队列合并

- 同一条记录的多个离线编辑合并成一个待同步草稿。
- 后一次编辑覆盖前一次同字段修改。
- 追加备注、追加子项等追加型操作可以保留为追加命令。
- 如果最后用户删除或取消这条记录，则压缩成删除或取消命令，前面的编辑不再同步。
- 离线新建后又离线删除，直接丢弃本地草稿，不同步。
- 离线新建后多次编辑，合并成一次创建请求。
- 离线新建后追加子项，也合并进创建请求。

## 系统可自动解决的冲突

系统可以自动处理天然可合并或重复提交无害的操作：

- 新增 Todo / 日程：同一 `clientOperationId` 不重复创建。
- 追加备注、追加子项：作为追加事件，不覆盖已有内容。
- 标记通知已读：重复标记仍然是已读。
- 稍后提醒：同一通知同一操作 ID 只创建一次稍后提醒。
- 完成 / 取消完成 Todo：如果服务端记录仍存在且用户有权限，可以基于最新版本再次提交。
- 低价值偏好：例如主题色、菜单固定，可以按最新版本自动重试。

需要用户处理的冲突：

- 编辑标题、描述、时间、地点、链接。
- 删除或取消。
- 服务端记录已删除或权限失效。
- 非白名单功能不进入移动端离线队列，离线时只提示联网后使用。

底线仍遵守 ADR 0016：删除、内容编辑、排序、布局和模块业务数据不能无提示覆盖。

## 用户可见状态

- 移动端 App 暂存成功时提示 `已暂存`，不要提示 `已保存`。
- 来源状态面板显示待同步数量，例如 `待同步 3 项`。
- 列表项使用轻量状态：`待同步`、`同步失败`、`有冲突`。
- Todo 完成这类高频操作中，首页小组件只显示小状态点或图标；完整 Todo 页面可显示更清楚的状态。
- 冲突处理不在通知中心完成。通知中心最多提示有离线修改需要处理，实际处理在对应模块页面或后续同步中心。
- 网络失败可以自动重试；权限失效、记录不存在和版本冲突不自动重试，进入待处理。

## 本地提交态

- 移动端 App 离线操作必须先写入本地数据库和待同步队列；本地落库成功后，界面才显示对应变化和 `待同步` 标识。
- 本地提交态只表示本设备已经记录操作，不表示服务端已经生效。
- 本地落库或写入队列失败时，不能提示 `已暂存`，应保留表单或操作上下文并提示失败。
- Todo 完成这类高频操作可以在本地落库成功后立即反映到列表，但必须带待同步图标或状态点。
- 删除、取消等高风险操作如果进入离线白名单，也必须显示待同步状态，不能让用户误以为服务端已经完成。
- 迁移、公开发布和权限变更不进入离线白名单，离线时提示联网后使用。
- 同步失败时本地状态保留但标记 `同步失败`，不静默回滚；用户可以重试或撤销本地修改。

## 离开、退出和切换账号

- 离开页面、切后台或关闭 App 只拦截未持久化的用户输入或操作。
- 如果操作已经持久化到服务端或本地离线队列，不拦截关闭。
- 有未保存表单草稿、正在写入队列但尚未确认落盘、或队列落盘失败时，需要关闭确认。
- 有待同步操作时，不允许直接退出登录或切换账号。
- 用户必须先同步、放弃本地待同步修改，或保留在本设备但账号锁定。
- 不支持把 A 账号的离线队列带到 B 账号。
- 不支持把某个来源的离线队列转移到另一个来源。

## 本地提醒兜底

- 服务端负责正式通知计划、收件箱、跨设备投递和已读状态。
- 移动端 App 缓存本设备近期提醒计划，用于弱网/无网时本地响铃、系统通知或本机横幅。
- 移动端 App 只缓存近期计划，不缓存无限远未来；窗口可配置。
- 本地提醒不写服务端收件箱，不改变服务端已读状态。
- 本地提醒触发后，客户端可以在本机记录已触发，避免同一设备重复响。
- 多设备可能同时有本地兜底提醒，可以接受；正式收件箱仍以后端为准。
- 重连后同步服务端计划：服务端取消则删除本地计划，服务端时间变更则更新本地计划，旧时间已本地触发的不撤回。
