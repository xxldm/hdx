## 公开方向摘要

本计划覆盖一组生活工具能力的产品方向：

- Todo：面向执行的轻任务，支持个人和共享空间场景。
- 日程事项：表示一件事在一段时间发生，不维护完成状态，可按用户设置提醒。
- 规则生成：用户维护候选池和规则，系统按周期自动生成抽取结果，并可落成 todo 或日程事项。
- 候选池：保存可抽取的候选项、标签、权重和可用时间条件。
- 通知中心：为 todo、日程、规则、计时器和后续模块提供统一通知能力。
- 公开主页与公开流：可信来源可以发布可订阅的公开日程或协作事项；普通用户默认没有对外发布权限。
- 协作事项：用户可通过邀请链接一起参与某件事，参与者拥有自己的个人日程、提醒和关联 todo。

## 前端页面与交互结论

### 首页组件总则

- 首页工作台采用可编辑布局；模块只提供可选组件，用户决定显示不显示、显示在哪里和占多大空间。
- 新账号默认布局只是启动建议，不是固定首页；用户可以删除、移动、缩放或重新添加组件。
- 第一版默认布局优先放 Todo、日程、计时器和日期倒计时等高频个人工具。
- Todo、日程、规则状态和公开流可以提供首页组件；候选池、公开发现、空间管理、实例管理不作为首页组件。
- 公开流可以做首页小组件；公开发现不进首页，只作为查找公开主页或事项的页面。
- 协作事项第一版不做首页一级组件；从关联 Todo / 日程、完整页面二级入口和邀请链接进入。
- 首页组件固定外部高度，内部滚动；首页主要用于查看和快速操作，不承载复杂配置、权限管理、完整历史或后台管理。

### 空间与成员

- 个人空间是账号默认空间，不支持解散；只有用户创建的共享空间可以解散。
- 共享空间第一版使用创建者、管理员、成员三类角色，不做只读成员。
- 空间创建者可以设置管理员、转让创建者和解散空间；空间管理员负责空间资料、成员和空间公共内容的日常管理。
- 普通成员第一版不默认创建空间公共 Todo、日程、规则或候选池；几个人临时一起做事走协作事项。
- 公司、部门、家庭、宿舍等都用平级空间表达，不做空间层级；需要全公司或部门范围时分别创建对应空间。
- 空间管理不进首页组件；从设置、空间上下文或相关资源入口进入。

### Todo

- 首页 Todo 组件固定高度，内部滚动；无时间 Todo 置顶且可折叠，组内优先使用用户手动排序，没有手动排序时按创建时间倒序；有时间 Todo 按临近程度排序。
- 首页组件支持展开子清单、勾选子项、完成/取消完成主 Todo 和快捷新增简单 Todo，不展示详情、操作记录、权限配置或来源详情。
- Todo 完整页面用于管理和编辑，筛选包括全部、今天、无时间、逾期、已完成、个人/空间和来源。
- Todo 列表按无时间、逾期、今天、未来、已完成等分组展示。
- 新增/编辑使用 `UModal`，删除确认使用 `UPopover`；第一版不做单独 Todo 详情页。
- Todo 列表项点击标题或行主体打开同一个 `UModal`；该 Modal 同时承担查看、编辑、新增表单，不另做只读详情页。
- Todo 列表内保留高频操作：完成/取消完成主 Todo、展开子清单、勾选/取消子项和打开编辑。
- Todo 只读、离线不可编辑、权限不足或来源不可写时，仍可用同一个 `UModal` 只读打开，并显示不能编辑的原因。
- Todo 取消或删除不设置缓冲期；删除表示取消并从普通列表移除，确认文案说明“删除后，这条内容会被取消，并从列表中移除”。
- 首页 Todo 组件比完整页面更轻，只保留快速新增、完成、子清单和打开 Modal；完整页面再提供筛选、分组、更多信息和更多操作。
- 空间 Todo 在列表和 `UModal` 中显示空间轻标签；有查看权限才显示，有完成权限才显示完成、取消完成和子清单勾选操作。
- 空间 Todo 的完成权限和查看权限绑定；没有查看权限的内容不进入列表，没有完成权限时只读展示并提示当前只能查看。
- 空间 Todo 的编辑、取消和删除入口只对管理员或具备对应管理权限的人显示；能完成的人可以追加个人备注。
- 逾期空间 Todo 如果子清单全部完成，状态按真实完成展示，不用虚拟状态覆盖真实状态。

### 日程事项

- 首页日程组件固定高度，内部滚动，按开始时间排序，进行中自然靠前并显示状态。
- 快捷新增默认只展示标题和时间区间，可展开填写描述、地点、链接和通知设置。
- 日程完整页面按进行中、未开始、已结束分组；默认隐藏已结束，提供“包含已结束”开关。
- 日程开始时间和结束时间必填；新增/编辑使用 `UModal`，删除确认使用 `UPopover`。
- 规则生成的日程标题、时间和来源摘要由规则控制；用户可改通知设置、追加备注、取消本次事项。取消后不允许重抽覆盖。
- 第一版不做单独日程详情页。
- 日程列表项点击标题或行主体打开同一个 `UModal`；该 Modal 同时承担查看、编辑、新增表单。
- 日程列表只展示高频扫描信息：标题、时间、进行中/未开始/已结束状态、地点/链接小图标、来源/空间轻标签。
- 日程取消或删除放在行内更多菜单里，确认使用 `UPopover`。
- 日程取消不设置缓冲期；取消后这件事立即变成已结束，未触发的开始提醒停止发送，已经收到的通知不撤回。
- 日程删除表示取消并从普通列表移除，删除不提供缓冲或普通恢复。
- 已结束日程默认不进入主列表；开启“包含已结束”后可查看，能否编辑继续按来源、权限和事项状态决定。
- 首页日程组件比完整页面更轻，只保留快速新增、打开 Modal 和核心状态；完整页面再提供分组、筛选、更多列信息和更多操作。

### 规则生成与候选池

- 候选池不做一级入口，可被多个规则复用，从规则编辑或二级入口进入。
- 首页规则组件只显示状态，例如成功、失败、已生成 Todo、已生成日程，不展示详情。
- 规则列表显示规则名称、周期、下一次生成时间和最近一次状态，支持启用/停用、手动执行、失败后手动重试和整体重抽。
- 规则编辑使用 `UModal`；删除确认使用 `UPopover`。
- 规则完整页面只做规则列表和操作入口；点击规则行打开规则 `UModal`。
- 规则 `UModal` 按区块组织：基本信息、生成内容、候选池、抽取槽位、权重与降权、最近结果。
- 基本信息区包含名称、启用状态、来源/空间和生成周期。
- 生成内容区包含生成 Todo 还是日程、标题模板、时间区间、描述模板和默认通知。
- 候选池区可以选择已有候选池，也可以进入当前弹窗内的候选池管理子视图。
- 候选池管理第一版不单独跳页，作为规则 `UModal` 内的子视图或二级 `UModal`；支持新增/编辑候选项、标签、权重、停用和删除。
- 最近结果区显示最近一次生成状态、生成出的 Todo / 日程入口、失败原因，以及重试或重抽入口。
- 规则时间区间必填，默认按周期填充，例如每日 0 点到 24 点、每周周一 0 点到周日 24 点。
- 候选项支持标签、基础权重、停用、删除；标签只作为匹配维度，不拆“分类”和“标签”两套体系。
- 候选池标签归属于候选池；规则选择候选池后，只使用该候选池内标签。
- 规则条件中正在使用的标签，必须先从规则中移除，才能停用或删除。
- 标签停用后不再作为新规则条件或新候选项标签可选项，但既有候选项、历史生成结果和用户可见摘要里继续保留标签显示。
- 标签删除后从可选列表移除；既有候选项、历史生成结果和用户可见摘要里保留可识别的历史标签显示，不同步移除。
- 停用标签和已删除标签需要有明显不同的视觉状态，避免用户误以为只是同一种置灰。
- 候选项停用表示暂时不参与后续抽取；候选项删除表示从候选池普通列表移除，但不影响历史生成结果摘要。
- 抽取槽位支持槽位名、包含标签多选、排除标签多选、数量、必需/可选。
- 全局包含/排除标签作为基础规则，具体槽位局部优先。
- 权重使用以 100 为基准的整数展示，例如 `150（近期命中 -50）`。
- 支持临时降权，例如近 3 次、1 周、1 月内降低 50 或 100。
- 删除规则和删除候选项使用 `UPopover` 确认。
- 手动执行、失败重试和整体重抽使用 `UModal` 确认；重抽确认需要说明不会覆盖已取消的本次事项。
- 重抽不默认排除当前结果，只使用当时最新规则、候选、权重和临时降权。
- 第一版不存候选快照；只保留生成结果、状态、失败原因、来源规则摘要和生成的 Todo / 日程关联。

### 通知中心

- 顶部通知按钮放在主题切换和用户头像之间，有未读时显示未读标记。
- 点击通知按钮打开通知面板；面板显示最近通知、稍后提醒、标记已读和查看详情，更多管理进入通知中心页面。
- 顶部通知面板使用轻量 `UPopover`，不跳页；面板固定宽高，内部滚动。
- 通知面板只放高频内容：最近通知列表、未读数、全部标记已读和进入通知中心。
- 通知面板里每条通知展示标题、来源轻标签、时间和简短正文。
- 通知面板支持单条标记已读、稍后提醒和打开关联内容。
- 完整通知中心页面负责管理和筛选，筛选包括全部、未读、稍后提醒、来源和通知类型。
- 完整通知中心列表按时间倒序展示；未来的稍后提醒可以单独分组或作为筛选结果展示。
- 完整通知中心支持批量标记已读；第一版不做批量删除，避免通知历史变成需要长期整理的第二套待办。
- 点击通知优先打开关联 Todo、日程或公开事项预览；来源不可用时只展示通知文本和不可用原因。
- 通知详情第一版不做单独页面，使用列表展开或 `UModal`。
- 通知中心保留未读/已读，因为它是收件箱语义；公开流不做未读/已读，只作为聚合查看。
- 收件箱只显示发了站内通知的内容；是否进入收件箱取决于是否发送站内通知，不按模块判断。
- 稍后提醒只做通知层延后，不修改业务时间；选项为 5 分钟、10 分钟、30 分钟。
- 通知设置支持全局默认渠道和事项级 default/custom；default 使用个人默认设置，custom 使用事项单独设置。
- 默认不强制站内兜底；用户没选渠道就不通知。提醒型事项至少选择一种渠道。
- 新事项设置时，没有权限的渠道不能勾选；老事项已开启但权限失效时，投递时记录跳过。
- 声音上传只在全局声音管理里做；事项里只能从已有声音列表选择。
- 设备静音开关是设备本地设置，只管理当前设备。静音后仅保留收件箱同步，不响声音、不弹横幅、不发系统或桌面通知。
- Web 可尝试使用浏览器 Notification API；Desktop/Windows 通过外壳接系统通知。

### 公开流、公开发现与公开主页

- 公开流只做首页小组件和公开事项预览，不做完整公开流页面。
- 公开流小组件固定高度，内部滚动；点击事项卡片打开公开事项 `UModal` 预览。
- 公开发现用于查找公开主页或公开事项，不包含加入、关注、取消、管理、隐藏、举报等操作。
- 公开发现普通点击不跳页、不新标签：点击主页结果打开公开主页 `USlideover` 预览，点击事项结果打开公开事项 `UModal` 预览。
- 公开主页有完整页面；只有用户在预览中明确点击“进入主页”时才切换到主页页面。
- Web 与客户端默认交互一致，不主动提供“新标签打开”主流程；浏览器原生中键、右键、Ctrl/Command 点击不刻意禁止。外部相关链接可以打开外部浏览器或新标签，并标识为外部链接。
- 公开事项第一版不做单独页面，只做预览弹窗；单独公开事项页作为后续预留。
- 公开事项 `UModal` 可以从公开流小组件、公开发现结果和公开主页事项列表打开。
- 公开事项 `UModal` 展示标题、时间、地点、描述、相关链接、发布主体、取消或结束状态。
- 公开事项 `UModal` 只放事项直接相关操作：加入个人日程、生成 Todo、查看发布主页。
- 从公开事项加入个人日程或 Todo 时写入默认保存位置；需要选择位置时使用已定义的保存位置选择 `UModal`。
- 公开事项已取消时禁用加入个人安排操作，并显示取消提示。
- 公开事项已结束后仍允许查看；第一版禁用加入个人安排，避免误加过期内容。
- 公开事项预览不做评论、收藏、转发、举报或隐藏。
- 公开主页 `USlideover` 可以从公开发现主页结果和公开事项里的发布主体打开。
- 公开主页 `USlideover` 展示主体名称、描述、相关链接和近期公开事项。
- 公开主页 `USlideover` 放关注/取消关注和进入主页完整页面操作。
- 公开主页 `USlideover` 只展示近期事项；完整主页页面再提供更多事项列表和“包含已结束”开关。
- 公开主页页面显示主体名称、公开描述、相关链接和公开事项列表；相关链接包含描述和地址，点击描述跳转地址。
- 公开主页默认显示未结束事项，已结束默认保留 7 天，并提供“包含已结束”开关。
- 公开主页不展示内部空间标签。
- 公开主页管理入口只对主页负责人和维护者显示；普通订阅者或访客只看到公开主页查看页。
- 有公开资格的账号可以创建或负责公开主页；公开主页负责人管理主页资料、相关链接、维护者、负责人转让和主页删除。
- 维护者第一版不编辑主页资料、不管理维护者、不删除主页；维护者只在被授权主页下发布和维护自己有权限的公开事项。
- 公开主页负责人转让第一版要求目标账号先成为维护者，再执行转让。
- 公开事项发布或重新公开时使用 `UModal` 二次确认，提示标题、时间、地点、描述和相关链接会公开。
- 公开事项管理页展示私有、公开、已结束、已取消公开等状态；取消公开后不可搜索。
- 公开事项取消公开时，需要通知已加入个人日程的人；下游个人日程和关联 Todo 取消，并保留标题帮助识别取消的是哪件事。
- 用户主动取消关注公开事项时，同时取消个人日程和关联 Todo，并停止接收后续更新。
- 如果用户只想取消自己的日程和 Todo，但继续关注源事项后续变更，需要从个人日程侧执行“取消本次安排”，关系退回仅关注变更。
- 公开事项删除等于取消并后台不可见；删除是高危操作，使用确认，不提供复制入口。
- 已结束事项允许重新公开；重新公开不发通知，只作为已订阅用户的数据更新。
- 取消公开和已结束事项可以分别设置在主页和公开流中的显示时长；主页已结束事项默认保留 7 天，公开流默认保留 1 天。
- 公开发现搜索页保留单一入口，顶部提供全部、主页、事项；全部结果先显示前 3 个匹配主页，再显示事项，继续翻页只显示事项。
- 公开流聚合同源协作事项投影时，可以做合并展示；是否合并作为用户显示开关，不改变源数据关系。

### 协作事项

- 协作事项第一版不做首页一级组件；从关联 Todo / 日程、完整页面二级入口和邀请链接进入。
- Todo 或日程关联协作事项时，在对应 `UModal` 或详情区域显示协作入口。
- 完整页面可以提供协作筛选或二级入口，用于查看自己参与或组织的协作事项。
- 邀请链接打开后进入协作事项预览/加入页，用户确认加入后再生成自己的个人日程、提醒和关联 Todo。
- 协作事项详情使用完整页面，不用 `UModal` 承载全部内容。
- 协作详情页按区块展示基本信息、我的安排、参与者、公共准备项、邀请与退出、公开发布。
- 基本信息展示标题、时间、地点、描述和相关链接。
- 我的安排展示我的个人日程、我的 Todo 和我的提醒设置；参与者只操作自己的个人安排和个人状态。
- 参与者列表只显示头像和公开名称；如果参与者有可访问的公开主页，头像可以点击进入。
- 公共准备项只作为协作事项内的共同清单展示，不自动塞进参与者个人 Todo。
- 组织者管理邀请链接；参与者退出后不再关注该协作事项，不提供继续关注选项。
- 协作事项列表只展示标题、时间、参与者摘要、我的状态和来源；复杂操作进入详情页。
- 组织者可以把协作事项公开到自己有权限的公开主页。
- 组织者公开协作事项时，可以同步扩散给有公开权限的参与者；参与者退出、组织者取消公开或组织者取消协作事项时，同步取消参与者侧公开。
