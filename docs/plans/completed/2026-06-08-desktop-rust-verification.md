## 目标

补齐 Desktop Tauri 骨架此前因 Rust 工具链缺失而留下的验证缺口，确认 Local/Online 两个 flavor 的 Rust 编译、Tauri 权限生成和 Desktop 质量门禁可以在当前 Windows 环境通过。

## 非目标

- 本轮不实现 `backend-all-in-one` sidecar 启动或打包。
- 本轮不实现本机 token 注入、远端地址持久化、系统通知、自启动、deep link、托盘、导入导出或 Windows wallpaper mode。
- 本轮不设计正式应用图标；仅补足 Tauri Windows 构建必需的占位 `icon.ico`。

## 验收标准

- `apps/desktop` 能通过 `cargo check --features flavor-local` 与 `--features flavor-online`。
- Tauri command permission 可以生成并被 `pnpm exec tauri permission ls` 识别。
- Desktop 质量门禁在不使用 `-NoBuild` 的情况下通过。
- 文档不再声称当前环境缺少 Rust 工具链，且保留真实剩余风险。

## 验证方式

- `cargo check --manifest-path src-tauri/Cargo.toml --features flavor-local`
- `cargo check --manifest-path src-tauri/Cargo.toml --features flavor-online`
- `pnpm exec tauri permission ls`
- `powershell -NoProfile -ExecutionPolicy Bypass -File scripts/quality-gate.ps1 -Scope desktop`
- `git -C apps/desktop diff --check`
- `git diff --check`

## 风险与阻塞

- `cargo`、`pnpm` 和 Tauri 命令需要访问用户目录缓存或构建产物，Codex 沙箱内可能失败；按权限规则走审批/提权路径。
- 首次 Rust 依赖解析需要联网访问 crates.io，网络不可用时只能保留 blocker。
